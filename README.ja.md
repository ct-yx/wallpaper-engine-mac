Open Wallpaper Engine（パッチ版）
=========

[English](README.md) | [简体中文](README.zh-CN.md) | [繁體中文](README.zh-TW.md) | **日本語**

[![GitHub license](https://img.shields.io/badge/license-GPL--3.0-blue.svg)](LICENSE)

[Open Wallpaper Engine](https://github.com/MrWindDog/wallpaper-engine-mac) のパッチフォークです。macOS 向けにシーン壁紙のレンダリングと Web 壁紙の修正を追加しています。

> **注意：** 本プロジェクトは Steam の商用版 Wallpaper Engine とは無関係です。Steam Workshop の壁紙アセットを表示できるオープンソースの macOS アプリケーションです。

> **プロジェクトの状態：** 本プロジェクトの開発は終了しました。より完全な macOS 実装については **[jipika/WaifuX](https://github.com/jipika/WaifuX)** をご覧ください。

## プロジェクトサイト

スクリーンショット、主な機能、最新のダウンロードは[プロジェクトサイト](https://ct-yx.github.io/wallpaper-engine-mac/)をご覧ください。

## ビルド済みリリース

最新の macOS ビルドは [Releases](../../releases) からダウンロードできます。リリースアーカイブは ad-hoc 署名のため、期限切れの Personal Team プロビジョニングプロファイルに依存しません。ただし Apple の公証は行われていないため、初回起動時に「プライバシーとセキュリティ」で開くことを確認する必要があります。

各リリースには、ダウンロードしたアーカイブを検証するための `.sha256` ファイルが含まれます。

## 0.9.10 の新機能

### シーンテキスト・サウンドと安全な SteamCMD インポート

- **静的なシーンテキスト** — シーン `text` オブジェクトの内容、元のフォント（パッケージ内フォントを含む）、ポイントサイズ、折り返し幅、色、アルファ、パディング、整列、親子階層、初期 `{"value": ...}` 設定値を描画します。
- **シーンのサウンドオブジェクト** — `sound` 配列と単一のサウンドパスを、展開済みファイルまたは `.pkg` アセットから再生します。`loop`、スリープ／復帰時の動作、共通の音量・再生速度コントロールに対応します。
- **完全なキャッシュ確認とロールバック** — `project.json` と再生可能なルートアセットの両方が存在する場合のみ SteamCMD コンテンツを再利用します。ステージング置換が成功するまで既存のローカルコピーを保持します。

## 0.9.9 の新機能

### SteamCMD セッションフロー

- **詳細画面でのセッション確認を廃止** — Workshop アイテムやダウンロード設定を開いても、保存済みログインを確認するだけでは SteamCMD を起動しません。
- **保存済みセッションから直接ダウンロード** — Steam へのログインが一度成功していれば、詳細画面にすぐ **Download** を表示します。SteamCMD がセッションを確認するのは実際にそのアイテムを転送するときだけです。
- **期限切れセッションを適切に復旧** — キャッシュ済みセッションの期限切れ時は SteamCMD を繰り返し再試行せず、ダウンロード設定が必要な状態に切り替えます。ダウンロード出力も同期して実際の失敗内容を保持します。

## 0.9.8 の新機能

### シーン変換とパーティクル寿命の再現性

- **パーティクル寿命オペレーター** — `sizechange`、`alphachange`、`colorchange` が作成者指定の寿命区間に従い、区間前後の値を保持して、Linux 参照レンダラーと同様に初期サイズ・アルファ・色へ乗算されます。
- **入れ子のシーン変換** — 画像とパーティクルが親グループ、元座標の原点、拡大縮小、角度、可視性、整列アンカーを保持し、すべてのレイヤーをシーンルートへ平坦化しません。
- **柔軟なシーンプロパティ** — 変換、色、サイズ、アルファ、可視性で `{"value": ...}` のカスタムラッパーをデコードします。`texture.png` のように明示的な画像拡張子を持つマテリアルテクスチャも直接読み込めます。

## 0.9.7 の新機能

### アニメーションするパーティクルテクスチャ

- **TEXS パーティクルをアニメーション** — パーティクルマテリアルが、先頭フレームだけでなく、デコード済みの TEXS マルチイメージ／アトラスフレームと `.tex-json` スプライトシートを再利用します。
- **元データのアニメーションモード** — ループ、`once`、`randomframe` のパーティクルモードを、作成者指定のフレーム時間と `sequencemultiplier` の速度でスケジュールします。
- **SpriteKit 互換のフォールバック** — SpriteKit は生存中の各パーティクルではなくエミッターごとのテクスチャを提供するため、フレーム選択は同じエミッターで共有されます。パーティクルの移動、寿命、既存のエミッター動作は変わりません。

## 0.9.6 の新機能

### SteamCMD キャッシュとライブラリの信頼性

- **ダウンロード済みコンテンツを即時再利用** — SteamCMD がすでに Workshop アイテムをダウンロード済みの場合、詳細画面はキャッシュされたファイルを直接インポートします。別の SteamCMD プロセスやログインセッションの復元は必要ありません。
- **設定済み Steam ライブラリをすべて検索** — 標準の macOS、Homebrew、単体 SteamCMD の場所に加え、インポーターは `libraryfolders.vdf` を読み取るため、2 番目または外部 Steam ライブラリのコンテンツも正しく見つけられます。
- **原子的なインポートを維持** — キャッシュ済み／新規ダウンロード済みの両方をステージングディレクトリ経由で置換し、コピー途中の壁紙がローカルライブラリに現れることを防ぎます。

## 0.9.5 の新機能

### TEXS シーンアニメーションテクスチャ

- **複数フレーム／アトラス TEXS 対応** — TEX パーサーがすべての TEXB イメージを走査し、ソース画像インデックス、表示時間、アトラス矩形を含む TEXS0001/0002/0003 フレームテーブルを読み取ります。
- **スプライトシートのサイドカー** — 静的な `materials/<texture>.tex-json` のスプライトシートシーケンスもシーン画像レイヤーで再生します。一般的な過去の 2 種類のサイドカー名に対応します。
- **ネイティブテクスチャ形式を拡張** — DXT1/DXT3/DXT5 と埋め込み PNG/JPEG に加え、raw RGBA/RGB、RG88、R8 TEX mipmap を描画します。非動画の TEXB0004 コンテナは上流の v3 互換 mipmap レイアウトに従います。
- **検証済みのバイナリ解析** — 画像、mipmap、フレーム記録に境界チェックを追加し、合成したマルチイメージ TEXS、アトラス TEXS、サイドカー、raw RGBA フィクスチャで検証しました。

## 0.9.4 の新機能

### シーン壁紙のマウスカメラパララックス

- **壁紙定義のパララックス** — `cameraparallax` を有効にしたシーン壁紙がポインターに追従します。レンダラーは元データの強さ、遅延、マウス影響、オブジェクトごとの `parallaxDepth` を読み取り、Workshop シーンで一般的な `{"user":…, "value":…}` ラッパーにも対応します。
- **レイヤー深度ごとの動き** — 画像レイヤーとパーティクルエミッターはそれぞれの初期位置から独立した深度に応じて移動します。パーティクルレイヤーは可視の動きを確保するため、上流実装の最小深度動作を維持します。
- **グローバル入力フックなし** — パララックスは SpriteKit のフレームループ内で更新し、アクティブな壁紙ビューのウィンドウ座標を使用するため、グローバルイベントモニターを追加しません。

## 0.9.3 の新機能

### シーンパーティクルとより堅牢なダウンロード

- **一般的なシーンパーティクルを描画** — SpriteKit が雨、雪、スパークルなどの一般的なボックス／球形エミッターを描画します。パーティクルテクスチャ、ブレンドモード、発生範囲、寿命、サイズ、速度、重力、アルファフェード、回転、標準的なインスタンスオーバーライドを対応付け、異常なパーティクルプールを制限します。
- **シーン座標を修正** — 静的画像レイヤーとパーティクルシステムは、Wallpaper Engine の左上原点座標を SpriteKit の左下原点座標へ変換します。
- **必要時だけキャッシュセッションを復元** — ダウンロード設定を開くと保存済みの SteamCMD セッションを自動確認します。Workshop の検索・詳細閲覧で SteamCMD が起動することはありません。
- **信頼できるライブラリ同期** — 対応する SteamCMD データディレクトリで正確な Workshop アイテムを探し、無期限のハングを回避してから、ステージングディレクトリ経由でローカルライブラリを置き換えます。

## 0.9.2 の新機能

### Workshop アクセスとシーンテクスチャ

- **先に閲覧、必要時にダウンロード設定** — Workshop の検索、フィルター、連続スクロール、詳細表示は Steam Web API のみで動作します。SteamCMD と Steam ログインは詳細画面でダウンロードを選択した時だけ必要です。

- **DXT シーンテクスチャ** — TEX パーサーが TEXB mipmap を読み取り、LZ4 圧縮を含む DXT1/BC1、DXT3/BC2、DXT5/BC3 テクスチャをソフトウェアでデコードします。

- **画面ごとの壁紙** — 各ディスプレイで異なる壁紙を独立して表示
- **有効/無効トグル** — モニターごとに壁紙のオン/オフを切り替え
- **自動検出** — 新しいモニターは接続時に自動的に検出・有効化

### マルチデスクトップ対応
壁紙がすべての macOS デスクトップ（Spaces）で連続再生されるようになりました。デスクトップ切り替え時も中断しません。

### 最近使った壁紙メニュー
ステータスバーメニューから壁紙を素早く切り替えられます。最近使用した10件の壁紙にワンクリックでアクセスできます。

### 再生設定 — 修正済み
パフォーマンス再生設定（他のアプリがフォーカスされた時の一時停止/ミュート/停止）がすべての壁紙タイプで正しく動作するようになりました。

### Steam Workshop ブラウザ
アプリ内から直接 Steam Workshop の壁紙を閲覧、検索、ダウンロードできます。
- **検索とフィルター** — 名前で検索、コンテンツレーティング（Everyone/Questionable/Mature）、タイプ（Scene/Video/Web）、ジャンルタグでフィルター
- **ソートオプション** — トレンド、最新、人気順、サブスクライブ数順
- **steamcmd 統合** — steamcmd を自動検出（Homebrew またはカスタムパス）、未インストール時はインストール手順を表示
- **Steam ログイン** — パスワード、Steam Guard、キャッシュセッション認証に対応
- **ダウンロード進捗表示** — リアルタイムステータス更新（認証中、ダウンロード %、検証、コピー中）
- **安全なデフォルト** — コンテンツレーティングを「Everyone」に設定し、成人向けコンテンツをフィルタリング

`.zip` ファイルから壁紙パッケージを直接インポート。手動解凍は不要です。ファイル > インポートおよびドラッグ＆ドロップに対応。

### 複数選択と一括解除
Cmd+クリックで複数の壁紙を選択し、右クリックで一括サブスクライブ解除。

### 壁紙ストレージの分離
壁紙は `~/Documents/Open Wallpaper Engine/` に保存されるようになり、Documents ディレクトリを直接使用しなくなりました。リポジトリをクローンした際の「error」壁紙を防止します。

## パッチ内容

### Web 壁紙 — グレー/空白レンダリングの修正
WebGL ベースの壁紙は `WKWebView` がローカルファイルアクセスをブロックしていたため、グレーの矩形として表示されていました。

**修正：** WKWebView 設定で `allowFileAccessFromFileURLs` と `allowUniversalAccessFromFileURLs` を有効にし、WebGL シェーダーがローカルテクスチャファイルを読み込めるようにしました。

### シーン壁紙 — ゼロから実装
シーン壁紙（Steam Workshop で最も一般的なタイプ）は完全に未実装で、「Hello, World!」のみ表示されていました。

**新しい実装：**
- **PKG パーサー** — Wallpaper Engine の PKGV アーカイブ形式を読み取り、scene.json、モデル、マテリアル、テクスチャを抽出
- **TEX パーサー** — TEXV0005 テクスチャコンテナを読み取り、JPEG/PNG 画像を抽出し、DXT1/DXT3/DXT5 mipmap をソフトウェアでデコード
- **Scene JSON デコーダー** — scene.json を解析、ポリモーフィックフィールド（値がプレーンタイプまたは `{"script":..,"value":..}` オブジェクト）を柔軟に処理
- **SpriteKit レンダラー** — シーン画像レイヤーを SKSpriteNode としてレンダリング、位置、サイズ、アルファ、カラーティント、ブレンドモードを正確に処理
- **プレビューフォールバック** — テクスチャを抽出できない場合は preview.jpg/png/gif にフォールバック
- **TEXI 形式検出** — テクスチャ形式と論理サイズを読み取り、圧縮 mipmap を正しく解析

### インポート — フォルダインポートの修正
インポートパネルが個別の壁紙フォルダと複数の壁紙を含む親ディレクトリの両方を正しく処理するようになりました。

## 現在の制限事項

- **パーティクルエフェクト** — 一般的な SpriteKit 互換エミッターと共有 TEXS／スプライトシートテクスチャアニメーションに対応しています。高度なパーティクル演算子、ロープ／トレイルレンダラー、コントロールポイント、パーティクル単位のフレーム選択は近似実装または未対応です。
- **オーディオリアクティブスクリプト** — Wallpaper Engine の JavaScript ベースのオーディオ視覚化スクリプトは実行されません。スクリプト付きプロパティは静的な `value` にフォールバックします。
- **シェーダーエフェクト** — カスタム GLSL シェーダー（ブルーム、ブラー、カラー補正）は適用されません。
- **カメラパララックス** — 一般的なオーソグラフィックシーンのマウスパララックスに対応しています。透視カメラ移動とカメラシェイクは未対応です。
- **アニメーションシーン** — シーン画像とパーティクルレイヤー上の TEXS マルチイメージ／アトラスアニメーションと `.tex-json` スプライトシートに対応しています。パーティクル単位のフレーム選択とタイムライン／スクリプト駆動のオブジェクトアニメーションは未対応です。
- **一部の JPEG サムネイル** — 少数の TEXB 形式 1 ファイルに macOS がデコードできない非標準 JPEG データが含まれています。

## ソースからビルド

### 前提条件
- macOS >= 13.0
- Xcode >= 14.4
- Xcode Command Line Tools

### 手順
```sh
git clone https://github.com/ct-yx/wallpaper-engine-mac.git
cd wallpaper-engine-mac
open "Open Wallpaper Engine.xcodeproj"
```

Xcode で署名証明書を自分のものに変更するか「Sign to Run Locally」を選択し、`Cmd + R` でビルド・実行します。

## 使い方

### Steam Workshop から閲覧・ダウンロード

1. **Workshop** タブに切り替え、表示されたら [Steam Web API キー](https://steamcommunity.com/dev/apikey) を入力
2. 検索・フィルター後に壁紙カードを開いて詳細を確認します。この操作に SteamCMD と Steam ログインは不要です
3. **Download** をクリックしてから steamcmd（`brew install steamcmd`）をインストールまたは既存バイナリを指定し、Wallpaper Engine を所有する Steam アカウントでログインします
4. 後続のキュー済みダウンロードではキャッシュ済み SteamCMD セッションを再利用します

### ローカルファイルからインポート

- **フォルダ：** ファイル > フォルダからインポート — `project.json` を含む壁紙フォルダを選択
- **Zip：** ファイル > インポート またはドラッグ＆ドロップで `.zip` ファイルを読み込み
- **手動：** 壁紙フォルダを `~/Documents/Open Wallpaper Engine/` に直接コピー
